# mtt-trend 시스템 아키텍처 개요

## 시스템 개요

mtt-trend는 52주 고점 테마 추세 분석 대시보드로, 한국 주식시장의 테마별 강도(RS: Relative Strength)를 실시간 추적합니다.

**코어 목표:**
- HTML 파싱을 통한 데이터 수집
- 테마별 주가 강도 분석
- 테마 트렌드 시각화 및 분석

## 주요 데이터 흐름

### 1. 데이터 수집 파이프라인
```
HTML 소스 (야후파이넌스 등)
    ▼
BeautifulSoup 파싱
    ▼
ingest.py CLI 처리
    ▼
SQLite 데이터 저장
    ▼
theme_daily, theme_stock_daily 테이블
```

### 2. API 요청-응답 흐름
```
프론트엔드 (React Query)
    ▼
API 호출 (useThemes, useStocks 훅)
    ▼
FastAPI 라우터
    ▼
SQLAlchemy 쿼리
    ▼
Pydantic 스키마 (응답 직렬화)
    ▼
JSON 응답
    ▼
프론트엔드 상태 업데이트 (TanStack Query 캐시)
    ▼
UI 렌더링 (Recharts 차트)
```

## 주요 설계 패턴

### 1. 계층 분리 (Layered Architecture)
- **프레젠테이션 계층**: Next.js 페이지 + React 컴포넌트
- **상태 관리 계층**: React Query (서버 상태), useState (UI 상태)
- **API 계층**: FastAPI 라우터 + Pydantic 스키마
- **비즈니스 로직 계층**: SQLAlchemy 쿼리 함수
- **데이터 접근 계층**: ORM 모델 (ThemeDaily, ThemeStockDaily)
- **영속성 계층**: SQLite 데이터베이스

### 2. CORS 정책
- 개발 환경: `localhost:3000`, `127.0.0.1:3000` 허용
- 프로덕션 환경: 추가 설정 필요

### 3. 캐싱 전략
- **React Query**: 자동 캐싱 및 배경 새로고침
- **데이터베이스 인덱스**: date, stock_name, theme_name 최적화

### 4. 검증 및 타입 안정성
- **백엔드**: Pydantic v2 검증 + 타입 힌트
- **프론트엔드**: TypeScript 인터페이스 + 타입 체크

## 시스템 경계 및 외부 인터페이스

### 외부 통신
1. **프론트엔드 → 백엔드**: HTTP/JSON (Axios)
2. **데이터 수집**: HTML 파싱 CLI (BeautifulSoup)
3. **데이터베이스**: SQLite (로컬 파일)

### 신뢰 경계
- CORS 검증 (프론트엔드 도메인)
- 입력 검증 (Pydantic 스키마)
- SQL 인젝션 방지 (SQLAlchemy ORM)

## 핵심 설계 결정사항

| 결정사항 | 선택 | 이유 |
|---------|------|------|
| 데이터베이스 | SQLite | 가볍고 배포 간단, 소규모 데이터셋에 적합 |
| 백엔드 프레임워크 | FastAPI | 자동 API 문서, 비동기 지원, 성능 우수 |
| 프론트엔드 프레임워크 | Next.js 14 | SSR/SSG 지원, 성능 최적화, TypeScript 우수 |
| 상태 관리 | React Query | 서버 상태 관리 최적화, 자동 캐싱 |
| 차트 라이브러리 | Recharts | 반응형, 접근성, 커스터마이징 용이 |
| 스타일링 | Tailwind CSS | 유틸리티 기반, 빠른 개발, 일관된 디자인 |
| 타입 시스템 | TypeScript + Pydantic | 점진적 타입 검증, 런타임 안정성 |

## 성능 최적화 전략

1. **데이터베이스**: 인덱스 활용 (date, stock_name, theme_name)
2. **API**: 쿼리 파라미터 필터링 (날짜, 임계값 등)
3. **프론트엔드**: React Query 캐싱, 컴포넌트 메모이제이션
4. **빌드**: Next.js 코드 스플리팅, 정적 최적화

## 확장성 고려사항

- **데이터 볼륨**: SQLite → PostgreSQL 마이그레이션 용이
- **사용자 증가**: 캐싱 계층 추가 (Redis), CDN 배포
- **기능 확장**: API 엔드포인트 모듈식 추가 가능
- **국제화**: 프론트엔드 다국어 지원 구조 (i18n) 준비 가능
