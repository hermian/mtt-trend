## F-01: 상위 테마 표시 개수 동적 설정

### AC-01-1: 기본값으로 상위 10개 테마 표시

```gherkin
Given 트렌드 페이지가 로드되었을 때
When TopThemesBar 컴포넌트가 렌더링되면
Then 상위 10개 테마가 avg_rs 내림차순으로 막대 차트에 표시된다
And 테마 개수 설정 컨트롤의 현재 값이 10으로 표시된다
```

### AC-01-2: 테마 개수 설정값 변경

```gherkin
Given TopThemesBar에서 테마 개수가 기본값 10으로 설정되어 있을 때
When 사용자가 테마 개수 설정값을 20으로 변경하면
Then 상위 20개 테마가 avg_rs 내림차순으로 막대 차트에 즉시 업데이트된다
And 설정 컨트롤의 현재 값이 20으로 표시된다
```

### AC-01-3: 설정 범위 제한

```gherkin
Given 테마 개수 설정 컨트롤이 화면에 표시되어 있을 때
When 사용자가 설정값을 조절하면
Then 최소값은 5, 최대값은 30으로 제한된다
And 범위를 벗어나는 값은 입력할 수 없다
```

### AC-01-4: 데이터 수 초과 방어 처리

```gherkin
Given 해당 날짜의 테마 데이터가 8개만 존재할 때
When 사용자가 테마 개수를 15로 설정하면
Then TopThemesBar는 8개 테마만 표시한다
And 에러나 경고 메시지가 발생하지 않는다
And 차트가 정상적으로 렌더링된다
```

## F-02: 2분할 가로 레이아웃 배치

### AC-02-1: 데스크탑 가로 배치

```gherkin
Given 화면 너비가 768px 이상인 데스크탑 환경에서
When 트렌드 페이지가 렌더링되면
Then TopThemesBar가 좌측 50%에 배치된다
And SurgingThemesCard가 우측 50%에 배치된다
And 두 컴포넌트가 동일한 행에 나란히 표시된다
```

### AC-02-2: 모바일 세로 스택

```gherkin
Given 화면 너비가 768px 미만인 모바일 환경에서
When 트렌드 페이지가 렌더링되면
Then TopThemesBar가 상단에 전체 너비로 표시된다
And SurgingThemesCard가 하단에 전체 너비로 표시된다
And 두 컴포넌트가 세로로 순차 배치된다
```

### AC-02-3: 반응형 전환

```gherkin
Given 데스크탑 환경에서 가로 배치로 표시되고 있을 때
When 브라우저 너비를 768px 미만으로 줄이면
Then 레이아웃이 세로 스택으로 자연스럽게 전환된다
And 콘텐츠 손실 없이 모든 데이터가 표시된다
```

## F-03: 동일 높이 유지

### AC-03-1: 콘텐츠 양 차이에도 동일 높이

```gherkin
Given 데스크탑 환경에서 가로 배치가 적용되었을 때
When TopThemesBar에 10개 테마, SurgingThemesCard에 3개 급등 테마가 표시되면
Then 두 컴포넌트의 외부 컨테이너 높이가 동일하다
And 콘텐츠가 적은 쪽은 나머지 공간이 자연스럽게 유지된다
```

### AC-03-2: 콘텐츠 초과 시 내부 스크롤

```gherkin
Given 데스크탑 환경에서 가로 배치가 적용되었을 때
When 한쪽 컴포넌트의 콘텐츠가 컨테이너 높이를 초과하면
Then 해당 컴포넌트 내부에 세로 스크롤이 활성화된다
And 다른 쪽 컴포넌트의 높이에 영향을 주지 않는다
And 스크롤바가 다크 테마에 맞게 표시된다
```
