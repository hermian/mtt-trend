## F-01: 페이지 로드 시 상위 5개 테마 자동 선택

### AC-01-1: 초기 로드 시 상위 5개 테마 자동 표시

```gherkin
Given 트렌드 페이지가 최초 로드되었을 때
When ThemeTrendChart 컴포넌트가 렌더링되면
Then 당일 기준 avg_rs 상위 5개 테마가 자동 선택된다
And 해당 5개 테마의 30일간 RS 추이가 라인 차트로 표시된다
And 각 테마의 Legend가 활성화 상태로 표시된다
```

### AC-01-2: 사용자 수동 변경 후 자동 선택 미적용

```gherkin
Given 페이지 로드 후 상위 5개 테마가 자동 선택되어 있을 때
When 사용자가 테마 선택을 수동으로 변경하면 (추가 또는 제거)
Then 변경된 선택이 유지된다
And 날짜나 소스 변경 시에도 사용자의 수동 선택이 유지된다
```

### AC-01-3: 테마 데이터 5개 미만인 경우

```gherkin
Given 당일 테마 데이터가 3개만 존재할 때
When 트렌드 페이지가 로드되면
Then 존재하는 3개 테마 전부가 자동 선택된다
And 차트에 3개 테마의 추이가 정상 표시된다
```

### AC-01-4: 테마 데이터가 없는 경우

```gherkin
Given 당일 테마 데이터가 0개일 때
When 트렌드 페이지가 로드되면
Then 빈 차트가 표시된다
And "데이터가 없습니다" 메시지가 차트 영역에 표시된다
```

## F-02: 기간 기본값 30일 명시

### AC-02-1: 기간 기본값 확인

```gherkin
Given 트렌드 페이지가 최초 로드되었을 때
When ThemeTrendChart의 기간 선택 UI를 확인하면
Then 30일이 기본 선택되어 있다
And 차트에 30일간의 데이터가 표시된다
```

### AC-02-2: 기간 변경 후 동작

```gherkin
Given ThemeTrendChart에서 기간이 30일로 설정되어 있을 때
When 사용자가 기간을 7일로 변경하면
Then 차트가 최근 7일간의 데이터로 업데이트된다
And 기간 선택 UI에 7일이 선택된 상태로 표시된다
```

## F-03: 라인 더블클릭 비활성화/활성화 토글

### AC-03-1: Legend 더블클릭으로 라인 비활성화

```gherkin
Given ThemeTrendChart에 5개 테마의 라인이 활성화 상태로 표시되어 있을 때
When 사용자가 "반도체" 테마의 Legend 항목을 더블클릭하면
Then "반도체" 라인의 opacity가 0.2로 변경된다
And "반도체" Legend 항목이 비활성화 상태로 표시된다 (연한 텍스트)
And 다른 4개 테마의 라인은 opacity 1.0을 유지한다
```

### AC-03-2: 비활성화된 라인 다시 더블클릭으로 복원

```gherkin
Given "반도체" 테마 라인이 비활성화(opacity 0.2) 상태일 때
When 사용자가 "반도체" Legend 항목을 다시 더블클릭하면
Then "반도체" 라인의 opacity가 1.0으로 복원된다
And "반도체" Legend 항목이 활성화 상태로 복원된다
```

### AC-03-3: 단일 클릭 동작 유지

```gherkin
Given ThemeTrendChart에 5개 테마 라인이 표시되어 있을 때
When 사용자가 Legend 항목을 단일 클릭하면
Then 기존 단일 클릭 동작이 정상적으로 수행된다
And 더블클릭 비활성화 동작이 트리거되지 않는다
```

### AC-03-4: 복수 테마 동시 비활성화

```gherkin
Given 5개 테마 라인이 모두 활성화 상태일 때
When 사용자가 3개 테마를 순차적으로 더블클릭하여 비활성화하면
Then 3개 테마의 라인이 opacity 0.2로 표시된다
And 나머지 2개 테마의 라인은 opacity 1.0을 유지한다
And 비활성화된 3개 테마의 Legend가 비활성 상태로 표시된다
```

## F-04: 단일 데이터 포인트 dot 표시

### AC-04-1: 신규 등장 테마 dot 표시

```gherkin
Given 테마 "AI반도체"가 조회 기간 내 마지막 날에만 데이터가 존재할 때
When ThemeTrendChart가 해당 테마를 렌더링하면
Then 해당 데이터 포인트 위치에 직경 8px의 둥근 dot이 표시된다
And dot의 색상은 해당 테마에 할당된 팔레트 색상이다
And 라인은 그려지지 않는다
```

### AC-04-2: dot 호버 시 툴팁 표시

```gherkin
Given 단일 데이터 포인트 dot이 차트에 표시되어 있을 때
When 사용자가 해당 dot에 마우스를 호버하면
Then 테마명, 날짜, RS 값이 포함된 툴팁이 표시된다
And 툴팁 형식은 다른 테마 라인의 툴팁과 동일하다
```

### AC-04-3: 다중 데이터 포인트 테마는 라인 유지

```gherkin
Given 테마 "반도체"에 30일간 데이터 포인트가 존재할 때
When ThemeTrendChart가 해당 테마를 렌더링하면
Then 기존과 동일하게 라인 차트로 표시된다
And 별도의 큰 dot은 표시되지 않는다 (기존 dot 스타일 유지)
```

### AC-04-4: 단일 포인트 테마와 다중 포인트 테마 공존

```gherkin
Given 선택된 5개 테마 중 1개가 단일 데이터 포인트, 4개가 다중 포인트일 때
When ThemeTrendChart가 렌더링되면
Then 1개 테마는 8px dot으로, 4개 테마는 라인으로 표시된다
And 모든 5개 테마의 Legend가 정상 표시된다
And 차트 축(X, Y)이 모든 데이터를 포함하여 올바르게 스케일링된다
```

## 엣지 케이스

| 케이스 | 기대 동작 |
|--------|-----------|
| 모든 선택 테마가 비활성화(더블클릭)된 경우 | 모든 라인이 opacity 0.2로 표시, 차트 축 유지 |
| 자동 선택된 5개 테마를 모두 수동 해제한 경우 | 빈 차트 표시, 재선택 가능 |
| 단일 포인트 테마를 더블클릭으로 비활성화한 경우 | dot의 opacity가 0.2로 변경 |
| 선택 테마 변경 후 이전 비활성화 상태 | 새로 선택된 테마는 활성화 상태, 이전 비활성화 상태 초기화 |
| 7일 기간에서 365일로 변경 시 단일 포인트 테마 | 365일에서는 데이터가 1개여도 dot 표시 |
| 빠른 연속 더블클릭 (3회 이상) | 토글 상태가 정확히 반영, 이벤트 무시 없음 |
