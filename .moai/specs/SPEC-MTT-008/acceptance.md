# SPEC-MTT-008 인수 기준: 테마 RS 추이 슬라이딩 윈도우 기간 설정

## 인수 기준

### AC-01: 초기 슬라이더 상태

```gherkin
Given 테마 RS 추이 차트 페이지가 로드되었을 때
Then 슬라이더가 기본값 30으로 설정되어 있어야 한다
And 슬라이더 라벨에 "기간: 30일"이 표시되어야 한다
And 범위 라벨에 "[7일 ~ 365일]"이 표시되어야 한다
And "30일" 프리셋 버튼이 활성 스타일(bg-blue-600 text-white)로 표시되어야 한다
And 나머지 프리셋 버튼(7일, 90일, 전체)은 비활성 스타일로 표시되어야 한다
And 차트가 30일 기간의 데이터로 렌더링되어야 한다
```

### AC-02: 슬라이더 드래그 동작

```gherkin
Given 슬라이더가 기본값 30으로 설정되어 있을 때
When 사용자가 슬라이더를 드래그하여 14로 변경하면
Then 슬라이더 라벨이 "기간: 14일"로 업데이트되어야 한다
And 모든 프리셋 버튼이 비활성 스타일로 표시되어야 한다 (14는 프리셋 값이 아님)
And 차트가 14일 기간의 데이터로 재조회 및 리렌더링되어야 한다
```

### AC-03: 프리셋 버튼 동기화

```gherkin
Given 슬라이더가 임의의 값(예: 14)으로 설정되어 있을 때
When 사용자가 "90일" 프리셋 버튼을 클릭하면
Then 슬라이더 값이 90으로 이동해야 한다
And 슬라이더 라벨이 "기간: 90일"로 업데이트되어야 한다
And "90일" 버튼이 활성 스타일로 표시되어야 한다
And 나머지 프리셋 버튼은 비활성 스타일로 표시되어야 한다
And 차트가 90일 기간의 데이터로 재조회 및 리렌더링되어야 한다
```

### AC-04: 전체 범위 프리셋

```gherkin
Given 슬라이더가 임의의 값으로 설정되어 있을 때
When 사용자가 "전체" 프리셋 버튼을 클릭하면
Then 슬라이더 값이 365로 이동해야 한다
And 슬라이더 라벨이 "기간: 365일"로 업데이트되어야 한다
And "전체" 버튼이 활성 스타일로 표시되어야 한다
And 차트가 365일 기간의 데이터로 재조회 및 리렌더링되어야 한다
```

### AC-05: 회귀 테스트 (기존 기능 보존)

```gherkin
Given 기간 슬라이더가 변경된 상태에서
Then 선택된 테마 목록은 변경 전과 동일해야 한다
And 테마 선택 disabled 상태는 보존되어야 한다
And 소스 토글(네이버/다음) 상태는 보존되어야 한다
And 차트 렌더링 형식(선 차트, 범례)은 동일해야 한다
```

## 엣지 케이스

### EC-01: 최소/최대 경계값

```gherkin
Given 슬라이더가 표시되어 있을 때
When 사용자가 슬라이더를 최소값(7)으로 이동하면
Then 슬라이더 값이 7 미만으로 내려가지 않아야 한다
And "7일" 프리셋 버튼이 활성 스타일로 표시되어야 한다
And 차트가 7일 데이터로 정상 렌더링되어야 한다

When 사용자가 슬라이더를 최대값(365)으로 이동하면
Then 슬라이더 값이 365를 초과하지 않아야 한다
And "전체" 프리셋 버튼이 활성 스타일로 표시되어야 한다
And 차트가 365일 데이터로 정상 렌더링되어야 한다
```

### EC-02: 빠른 연속 슬라이더 조작

```gherkin
Given 슬라이더가 표시되어 있을 때
When 사용자가 슬라이더를 빠르게 연속으로 조작하면 (예: 30 -> 50 -> 100 -> 200)
Then React Query의 dedupe 메커니즘에 의해 불필요한 중간 API 호출이 방지되어야 한다
And 최종 슬라이더 값에 해당하는 차트 데이터가 정상 표시되어야 한다
And UI가 프리즈되거나 지연되지 않아야 한다
```

### EC-03: 프리셋 버튼 빠른 연속 클릭

```gherkin
Given 프리셋 버튼이 표시되어 있을 때
When 사용자가 서로 다른 프리셋 버튼을 빠르게 연속 클릭하면 (예: 7일 -> 90일 -> 전체)
Then 최종 클릭된 버튼의 값으로 슬라이더가 동기화되어야 한다
And 최종 값에 해당하는 차트 데이터가 정상 표시되어야 한다
```

## 툴팁 검증

### TC-01: 툴팁 표시

```gherkin
Given 기간 라벨이 표시되어 있을 때
When 사용자가 기간 라벨에 마우스를 hover하면
Then 툴팁이 표시되어야 한다
And 툴팁 내용에 "RS 추이 조회 기간 (7~365일)"이 포함되어야 한다
And 툴팁 내용에 "기간이 길수록 더 많은 과거 데이터를 표시합니다"가 포함되어야 한다

When 사용자가 기간 라벨에서 마우스를 벗어나면
Then 툴팁이 사라져야 한다
```
