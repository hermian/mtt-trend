# Acceptance Criteria: 모바일 반응형 사이드바 레이아웃

## 1. 테스트 시나리오

### 1.1 Scenario 1: 모바일 뷰포트 사이드바 너비

**Given** 사용자가 모바일 기기(너비 < 640px)로 페이지에 접속하고
**And** 사이드바가 확장된 상태(collapsed = false)이며
**When** 페이지가 로드되면
**Then** 사이드바 너비는 64px이어야 한다
**And** 사이드바는 화면 너비의 20% 이하를 차지해야 한다

```gherkin
Scenario: 모바일 뷰포트 사이드바 너비 검증
  Given 사용자가 375px 너비의 모바일 기기로 접속함
  And 사이드바가 확장된 상태임
  When 페이지가 완전히 로드됨
  Then 사이드바의 computed width는 64px이어야 함
  And 사이드바 점유율은 17.1%이어야 함
```

---

**When** 페이지가 로드되면

```gherkin
Scenario: 소형 태블릿 뷰포트 사이드바 너비 검증

  And 사이드바가 확장된 상태임
  When 페이지가 완전히 로드됨
  Then 사이드바의 computed width는 80px이어야 함
```

---

### 1.4 Scenario 4: 데스크톱 뷰포트 사이드바 너비

**Given** 사용자가 데스크톱(너비 >= 1024px)으로 페이지에 접속하고
**And** 사이드바가 확장된 상태(collapsed = false)이며
**When** 페이지가 로드되면
**Then** 사이드바 너비는 224px이어야 한다
**And** 기존 동작과 동일해야 한다

```gherkin
Scenario: 데스크톱 뷰포트 사이드바 너비 검증
  Given 사용자가 1024px 너비의 데스크톱으로 접속함
  And 사이드바가 확장된 상태임
  When 페이지가 완전히 로드됨
  Then 사이드바의 computed width는 224px이어야 함
  And 메뉴 텍스트가 완전히 표시되어야 함
```

---

### 1.5 Scenario 5: Collapse 기능 동작

**Given** 사용자가 임의의 뷰포트 크기로 페이지에 접속하고
**And** 사이드바가 확장된 상태이며
**When** 사용자가 collapse 버튼을 클릭하면
**Then** 사이드바 너비는 64px로 축소되어야 한다
**And** 아이콘만 표시되어야 한다

```gherkin
Scenario: Collapse 기능 동작 검증
  Given 사용자가 임의의 뷰포트로 접속함
  And 사이드바가 확장된 상태임
  When collapse 버튼을 클릭함
  Then 사이드바의 computed width는 64px이어야 함
  And collapsed 상태가 true여야 함
  And 메뉴 텍스트가 숨겨져야 함
```

---

### 1.6 Scenario 6: 뷰포트 리사이즈 시 반응형 전환

**Given** 사용자가 데스크톱(1024px)에서 페이지에 접속하고
**And** 사이드바 너비가 224px이며
**When** 브라우저 창을 모바일 크기(375px)로 리사이즈하면
**Then** 사이드바 너비는 64px로 변경되어야 한다
**And** 전환은 부드럽게 이루어져야 한다

```gherkin
Scenario: 뷰포트 리사이즈 시 반응형 전환 검증
  Given 사용자가 1024px 너비로 접속함
  And 사이드바가 224px 너비임
  When 브라우저 창을 375px로 리사이즈함
  Then 사이드바 너비는 64px로 변경되어야 함
  And 전환 애니메이션이 적용되어야 함
```

| 768px       | 96px               | 대기        |
| 1023px      | 96px               | 대기        |
| 1024px      | 224px              | 대기        |

### 2.2 EC-02: Collapse 상태에서 리사이즈

**Given** 사이드바가 collapse 상태(64px)이고
**When** 뷰포트를 리사이즈하면
**Then** 사이드바 너비는 64px로 유지되어야 한다

### 2.3 EC-03: 고해상도 디스플레이

**Given** 사용자가 4K 디스플레이(3840px)로 접속하면
**When** 페이지가 로드되면
**Then** 사이드바 너비는 224px이어야 한다
**And** 콘텐츠 영역이 적절히 확장되어야 한다

### 2.4 EC-04: 세로 모드 vs 가로 모드

**Given** 모바일 기기에서
**When** 세로 모드(375px)에서 가로 모드(667px)로 전환하면
**Then** 사이드바 너비는 64px에서 80px로 변경되어야 한다
