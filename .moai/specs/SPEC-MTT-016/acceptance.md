# SPEC-MTT-016 인수 기준

## TAG: SPEC-MTT-016

---

## 인수 기준 (Acceptance Criteria)

### AC-001: Viewport 메타데이터 올바른 형식 (R-001)

**Given** Next.js 16 앱이 빌드되었을 때
**When** 빌드된 HTML을 확인하면
**Then** `<meta name="viewport" content="width=device-width, initial-scale=1">` 태그가 정상적으로 포함되어야 한다
**And** Next.js 빌드 로그에 viewport 관련 경고가 없어야 한다

---

### AC-002: PC 레이아웃 완전 보존 (R-005)

**Given** PC Chrome 브라우저에서 1920x1080 해상도로 접속했을 때
**When** `/trend` 페이지를 로드하면
**Then** 사이드바가 왼쪽에 고정 표시되어야 한다 (w-56 확장, w-16 접힘)
**And** 사이드바 접기/펼치기 버튼이 정상 동작해야 한다
**And** 메인 콘텐츠 영역의 패딩이 24px(p-6)이어야 한다
**And** 모바일 헤더(햄버거 메뉴)가 표시되지 않아야 한다
**And** 모든 차트, 테이블, 카드 레이아웃이 기존과 동일해야 한다

---

### AC-003: 모바일 사이드바 기본 숨김 (R-002)

**Given** Android Chrome 브라우저에서 360px 너비로 접속했을 때
**When** `/trend` 페이지를 로드하면
**Then** 사이드바가 화면에 보이지 않아야 한다
**And** 메인 콘텐츠가 화면 전체 너비를 사용해야 한다
**And** 가로 스크롤이 발생하지 않아야 한다

---

### AC-004: 모바일 헤더 표시 (R-006)

**Given** 360px 너비 모바일 화면에서 접속했을 때
**When** 페이지가 로드되면
**Then** 화면 상단에 모바일 헤더가 표시되어야 한다
**And** 모바일 헤더에 햄버거 메뉴 아이콘과 "MTT Trend" 텍스트가 포함되어야 한다
**And** 모바일 헤더가 스크롤 시에도 고정 위치를 유지해야 한다

**Given** 768px 이상 화면에서 접속했을 때
**When** 페이지가 로드되면
**Then** 모바일 헤더가 표시되지 않아야 한다

---

### AC-005: 모바일 오버레이 사이드바 열기 (R-003)

**Given** 모바일 화면에서 사이드바가 닫혀있을 때
**When** 햄버거 메뉴 버튼을 탭하면
**Then** 반투명 검정 오버레이 배경이 나타나야 한다
**And** 사이드바가 왼쪽에서 슬라이드 인 애니메이션으로 나타나야 한다
**And** 사이드바 내 네비게이션 항목이 모두 표시되어야 한다

---

### AC-006: 모바일 오버레이 사이드바 닫기 (R-003)

**Given** 모바일 화면에서 오버레이 사이드바가 열려있을 때
**When** 오버레이 배경(사이드바 외부)을 탭하면
**Then** 사이드바와 오버레이가 닫혀야 한다

**Given** 모바일 화면에서 오버레이 사이드바가 열려있을 때
**When** 네비게이션 항목("52주 트렌드")을 탭하면
**Then** 해당 페이지로 이동하고 사이드바가 자동으로 닫혀야 한다

---

### AC-007: 모바일 패딩 최적화 (R-004)

**Given** 360px 너비 모바일 화면에서 `/trend` 페이지를 볼 때
**When** 메인 콘텐츠 영역을 확인하면
**Then** 패딩이 12px(p-3)로 적용되어야 한다
**And** 섹션 간 간격이 16px(space-y-4)로 적용되어야 한다

**Given** 768px 이상 화면에서 `/trend` 페이지를 볼 때
**When** 메인 콘텐츠 영역을 확인하면
**Then** 패딩이 24px(p-6)로 유지되어야 한다
**And** 섹션 간 간격이 24px(space-y-6)로 유지되어야 한다

---

### AC-008: 빌드 및 테스트 통과

**Given** 모든 코드 변경이 적용된 후
**When** 다음 명령어를 실행하면
**Then** 모두 성공해야 한다:
- `pnpm build` -- 빌드 성공, 에러 없음
- `pnpm lint` -- 린트 경고/에러 없음
- `pnpm test` -- 기존 테스트 모두 통과
