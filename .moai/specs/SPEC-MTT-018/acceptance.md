# SPEC-MTT-018 수락 기준: 사용자 가이드 문서 생성 및 HTML 웹 서비스

## 1. 테스트 시나리오

### 시나리오 1: UserGuide.md 문서 존재 및 구조 검증

```gherkin
Given 프로젝트 루트 디렉토리에 접근할 수 있다
When UserGuide.md 파일을 확인한다
Then 파일이 프로젝트 루트(`/UserGuide.md`)에 존재해야 한다
And 다음 섹션 헤딩이 포함되어야 한다:
  | 섹션 |
  | 소개 |
  | 시작하기 |
  | 주요 기능 사용법 |
  | FAQ |
  | 문제 해결 |
And 주요 기능 사용법 섹션에 7개 기능이 모두 기술되어야 한다:
  | 기능 |
  | 데이터 소스 전환 |
  | 테마별 일일 RS 집계 |
  | 급등 테마 탐지 |
  | 테마 RS 시계열 추이 |
  | 지속 강세 종목 탐지 |
  | 그룹 액션 탐지 |
  | 교집합 추천 |
And 문서 전체가 한국어로 작성되어야 한다
```

**관련 요구사항:** REQ-MTT-018-01, REQ-MTT-018-02, REQ-MTT-018-03, REQ-MTT-018-04

### 시나리오 2: 가이드 웹 페이지 접근 및 렌더링

```gherkin
Given 프론트엔드 개발 서버가 실행 중이다
When 사용자가 브라우저에서 http://localhost:3000/guide 에 접속한다
Then 사용자 가이드 콘텐츠가 HTML로 렌더링되어 표시되어야 한다
And 기존 대시보드와 동일한 사이드바와 헤더가 표시되어야 한다
And Markdown 콘텐츠가 적절한 타이포그래피 스타일로 렌더링되어야 한다
And 테이블, 코드 블록, 리스트 등 GFM 요소가 정상적으로 표시되어야 한다
```

**관련 요구사항:** REQ-MTT-018-05, REQ-MTT-018-06, REQ-MTT-018-07

### 시나리오 3: 목차(TOC) 동작 검증

```gherkin
Given 사용자가 /guide 페이지에 접속해 있다
When 페이지가 로드된다
Then 페이지 상단 또는 사이드에 목차가 표시되어야 한다
And 목차 항목은 문서의 헤딩(##, ###)과 일치해야 한다

When 사용자가 목차의 특정 항목을 클릭한다
Then 해당 섹션으로 스무스 스크롤이 발생해야 한다
```

**관련 요구사항:** REQ-MTT-018-08

### 시나리오 4: 사이드바 네비게이션 통합

```gherkin
Given 사용자가 대시보드의 아무 페이지에 접속해 있다
When 사이드바를 확인한다
Then "사용자 가이드" 메뉴 항목이 표시되어야 한다

When 사용자가 "사용자 가이드" 메뉴를 클릭한다
Then /guide 페이지로 이동해야 한다
And 사이드바에서 "사용자 가이드" 항목이 활성화 상태로 표시되어야 한다
```

**관련 요구사항:** REQ-MTT-018-09, REQ-MTT-018-10

### 시나리오 5: 모바일 반응형 검증

```gherkin
Given 사용자가 모바일 기기(화면 너비 < 768px)에서 /guide 에 접속한다
When 페이지가 로드된다
Then 콘텐츠가 화면 너비에 맞게 반응형으로 표시되어야 한다
And 텍스트가 가독성 있는 크기로 표시되어야 한다
And 테이블이 가로 스크롤 가능해야 한다
And 모바일 헤더의 햄버거 메뉴에서 "사용자 가이드"에 접근 가능해야 한다
```

**관련 요구사항:** REQ-MTT-018-12

### 시나리오 6: README.md 업데이트 검증

```gherkin
Given README.md 파일을 확인한다
When "사용자 가이드" 관련 내용을 검색한다
Then 사용자 가이드에 대한 설명이 포함되어야 한다
And /guide 경로에 대한 안내가 포함되어야 한다
```

**관련 요구사항:** REQ-MTT-018-11

## 2. 엣지 케이스

| 케이스 | 예상 동작 |
|--------|----------|
| UserGuide.md 파일이 누락된 경우 | 가이드 페이지에 "가이드 문서를 찾을 수 없습니다" 안내 메시지 표시 |
| UserGuide.md가 빈 파일인 경우 | 빈 콘텐츠 영역 + 목차 미표시 |
| 매우 긴 문서 (1000줄 이상) | 목차를 통한 빠른 탐색 가능, 성능 저하 없음 |
| Markdown에 잘못된 문법이 있는 경우 | 렌더링 가능한 부분까지 표시, 깨진 부분은 원문 텍스트로 표시 |
| 외부 이미지 링크가 포함된 경우 | 이미지 정상 로드 또는 alt 텍스트 표시 |
