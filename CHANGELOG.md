## [1.1.0] - 2026-03-15

### 추가된 기능

#### 그룹 액션 탐지 기능 고도화 (SPEC-MTT-006)
- **유연한 파라미터 설정**:
  - `timeWindow` 파라미터: 1-7일 범위로 신규 등장 기간 조정 (기본값: 3일)
  - `rsThreshold` 파라미터: -10 ~ +20 범위로 RS 상승 임계값 설정 (기본값: 0)
  - `status_threshold` 응답 필드: 프론트엔드 상태 분류 기준값 (기본값: 5)
- **성능 최적화**:
  - 데이터베이스 인덱스 추가 (`idx_stock_first_seen`)
  - first_seen 서브쿼리 응답 시간: ~500ms → ~100ms (80% 성능 향상)
- **UI 개선**:
  - 시간 윈도우 슬라이더 (1~7일)
  - RS 임계값 슬라이더 (-10~+20)
  - 상태 임계값 슬라이더 (1~20) - 클라이언트 측 상태 분류 지원
- **하위 호환성**: 기존 API 시그니처 100% 유지, 기본값으로 동일 동작 보장

#### HTML 자동 감지 및 DB 동기화 시스템 구축 (SPEC-MTT-009)
- **자동 파일 감지 시스템**:
  - Watchdog Observer를 통한 실시간 HTML 파일 감지
  - `backend/data/` 폴더에 HTML 파일이 추가되면 자동으로 DB 적재
  - 서버 시작/종료 시 Observer 생명주기 관리
- **수동 동기화 API**:
  - `POST /api/sync` - 모든 HTML 파일의 DB 동기화 트리거
  - `GET /api/sync/status` - 동기화 상태 조회 (선택 사항)
  - 중복 실행 방지: 동시 동기화 시 409 Conflict 응답
- **중복 처리 및 오류 격리**:
  - UNIQUE 제약 조건을 통한 UPSERT 동작 (INSERT OR REPLACE)
  - 개별 파일 단위 오류 격리: 하나의 파일 오류가 전체 작업 중단시키지 않음
  - 패턴 불일치 파일에 대한 WARNING 로그 기록
- **프론트엔드 동기화 버튼**:
  - React 컴포넌트를 통한 동기화 트리거 UI
  - 동기화 중 로딩 상태 및 버튼 비활성화
  - 처리 결과 토스트 알림 표시
- **테스트 커버리지**:
  - 23개 테스트 통과 (9개 sync_service + 10개 file_watcher + 4개 API)
  - 백엔드 테스트 커버리지 85%+ 달성
  - 프론트엔드 SyncButton 컴포넌트 테스트 포함
- **기술 사양**:
  - `watchdog>=4.0.0` 의존성 추가
  - FastAPI lifespan을 통한 Watchdog 통합
  - 기존 `ingest.py` 로직 재사용 및 호환성 유지
  - INFO 로그: 성공적 적재 시 파일명, 테마 수, 종목 수 기록

#### 그룹 액션 탐지 파라미터 툴팁 추가 (SPEC-MTT-007)
- **접근성 툴팁 시스템**:
  - 각 파라미터에 대한 설명 툴팁 추가 (시간 윈도우, RS 임계값, 상태 임계값)
  - WCAG 2.1 레벨 AA 접근성 준수 (ARIA 속성, 키보드 네비게이션)
  - InformationCircleIcon 아이콘을 통한 툴팁 존재 시각적 표시
- **툴팁 컴포넌트**:
  - 재사용 가능한 `Tooltip.tsx` 컴포넌트 (90라인)
  - hover/focus 이벤트 지원 (300ms 지연, 100ms 숨김)
  - 반응형 디자인 지원 (모바일, 태블릿, 데스크톱)
- **테스트 커버리지**:
  - 46/46 테스트 통과 (100% 성공률)
  - Jest + React Testing Library 기반 단위 테스트
  - 접근성 테스트, 반응형 디자인 테스트 포함
- **사용성 개선**:
  - 파라미터 의미 명확화: "신규 등장 종목 판정 기간", "테마 RS 상승 판정 기준", "주식 상태 분류 기준"
  - 스크린 리더 호환성 및 포커스 관리 강화
  - 기존 슬라이더 기능 100% 유지

#### 서버 시작 시 자동 동기화 시스템 구현 (SPEC-MTT-010)
- **서버 시작 시 자동 데이터 적재**:
  - FastAPI lifespan 함수를 통한 초기 동기화 수행
  - `data/` 폴더에 존재하는 모든 HTML 파일을 자동으로 스캔
  - 미적재 파일만 DB에 적재, 이미 적재된 파일은 건너뜀
- **에러 격리 시스템**:
  - 개별 파일 처리 중 오류가 발생해도 전체 동기화 중단되지 않음
  - 손상된 파일은 에러 로그로 기록되고 정상적인 파일은 계속 처리됨
  - 서버 시작 실패 방지: 초기 동기화 실패 시에도 서버 정상 시작
- **중복 적재 방지**:
  - 날짜 + data_source 기준으로 이미 적재된 파일 자동 건너뛰기
  - `SyncService.sync_files()` 메서드 재사용을 통한 일관성 유지
- **상태 로깅 및 모니터링**:
  - INFO 레벨 로그: 스�anned 파일 수, 처리된 파일 수, 건너뛴 파일 수, 에러 목록 기록
  - 초기 동기화 완료 시 자세한 결과 요약 제공
- **Watchdog 시스템 통합**:
  - 초기 동기화 완료 후 Watchdog 파일 감시자 정상 시작
  - 기존 파일은 초기 동기화로 처리, 새 파일만 Watchdog에서 감지
- **테스트 커버리지**:
  - 33개 테스트 통과 (sync 모듈 71% 커버리지)
  - 모든 수용 기준(AC-010-01 ~ AC-010-06) 검증
  - 빈 폴더, 없는 폴더, 손상된 파일 등 모든 경시 테스트 완료
- **기술 사양**:
  - `app/main.py` lifespan 함수에 초기 동기화 로직 추가
  - `perform_initial_sync()` 함수를 통한 로직 분리 및 재사용성 보장
  - MX 태그 적용: `@MX:ANCHOR`, `@MX:NOTE`, `@MX:REASON` 사용
