## 소개

**테마 트렌드 대시보드**는 한국 주식 시장의 신고가 달성 종목과 MTT(Mark Minervini Template) 종목들을 테마별로 분석하여 장 추세를 발견하는 실시간 분석 도구입니다.

### 이 대시보드가 해결하는 문제

- **정보 분산**: 장중 신고가 달성 종목 수십 개를 개별 추적하기 어려운 문제
- **테마 파악**: 신고가 종목들이 속한 테마(산업/트렌드)를 한눈에 파악하기 어려운 문제
- **추세 감지**: 강세 테마의 시작과 약화 시점을 정량적으로 측정하기 어려운 문제

### 핵심 지표: RS(상대강도) 점수

RS(Relative Strength) 점수는 특정 테마에 속한 종목들이 얼마나 강세를 보이는지를 나타내는 지표입니다. 점수가 높을수록 해당 테마의 시장 내 강도가 강함을 의미합니다.

## 시작하기

### 접속 방법

로컬 환경에서 실행 중인 경우:

- **프론트엔드 대시보드**: http://localhost:3000
- **백엔드 API 문서**: http://localhost:8000/docs

### 인터페이스 개요

대시보드는 크게 세 영역으로 구성됩니다.

**사이드바 (좌측)**

사이드바에서 주요 메뉴를 선택할 수 있습니다.

- `52주 트렌드`: 메인 분석 페이지 (테마별 RS 집계, 신규 급등 탐지, 시계열 분석 등)
- `사용자 가이드`: 현재 페이지

**메인 영역 (중앙)**

선택한 메뉴의 내용이 표시됩니다. 52주 트렌드 페이지에는 다음 섹션들이 포함됩니다.

- 데이터 소스 전환 버튼 (우측 상단)
- 테마별 일일 RS 집계 테이블
- 신규 급등 테마 탐지 패널
- 테마 RS 시계열 차트
- 지속 강세 종목 탐지 패널
- 그룹 액션 탐지 패널
- 교집합 추천 패널

**모바일 환경**

화면 너비가 좁은 모바일 기기에서는 좌측 상단의 햄버거 메뉴 아이콘을 탭하여 사이드바를 열 수 있습니다.

### 데이터 소스 전환

페이지 우측 상단에 **데이터 소스 토글 버튼**이 있습니다.

- **52주 신고가**: 52주 신고가를 달성한 종목 기반 분석
- **MTT 종목**: Mark Minervini Template 패턴을 만족하는 종목 기반 분석

토글 버튼 클릭 시 전체 페이지의 데이터가 선택한 소스로 전환됩니다.

## 주요 기능 사용법

### 1. 데이터 소스 전환

**위치**: 52주 트렌드 페이지 우측 상단

**사용법**

1. 페이지 우측 상단의 토글 버튼을 확인합니다.
2. `52주 신고가` 또는 `MTT 종목` 중 원하는 데이터 소스를 클릭합니다.
3. 페이지 전체의 데이터가 선택한 소스로 즉시 전환됩니다.

**두 소스의 차이**

| 항목 | 52주 신고가 | MTT 종목 |
|------|------------|---------|
| 기준 | 최근 52주 내 최고가 달성 | Mark Minervini 기술적 패턴 만족 |
| 특성 | 모멘텀 강세 종목 | 기술적 분석 기반 종목 |
| 활용 | 장 추세 파악 | 기술적 진입점 탐색 |

---

### 2. 테마별 일일 RS 집계 조회

**위치**: 52주 트렌드 페이지 상단 테이블

**사용법**

1. 날짜 선택기에서 조회할 날짜를 선택합니다 (기본값: 가장 최근 데이터 날짜).
2. 테이블에 해당 날짜의 테마별 RS 점수가 내림차순으로 표시됩니다.

**테이블 컬럼 설명**

| 컬럼 | 설명 |
|------|------|
| 테마명 | 종목들이 속한 테마(산업/트렌드) 이름 |
| RS 점수 | 해당 테마의 상대강도 점수 (높을수록 강세) |
| 종목 수 | 해당 테마에 속한 종목 수 |
| 평균 등락률 | 테마 구성 종목들의 평균 등락률 |
| 전일 대비 | RS 점수의 전일 대비 변화량 |

**활용 팁**

- RS 상위 테마가 오늘의 장 주도 테마입니다.
- 전일 대비 변화량이 큰 테마는 모멘텀 급변이 일어난 테마입니다.

---

### 3. 신규 급등 테마 탐지

**위치**: 52주 트렌드 페이지 중간 섹션

**사용법**

1. 탐지 임계값을 설정합니다 (기본값: +10). 이 값은 "5일 이동평균 대비 RS가 얼마나 상승했을 때 급등으로 판단할지"를 결정합니다.
2. 조회 날짜를 선택합니다.
3. 설정한 임계값 이상으로 RS가 상승한 테마 목록이 표시됩니다.

**임계값 가이드**

| 임계값 | 의미 |
|--------|------|
| +5 이상 | 약한 신호 (노이즈 포함 가능성 높음) |
| +10 이상 | 적정 신호 (기본 추천값) |
| +15 이상 | 강한 신호 (확실한 급등 테마만 탐지) |

### 4. 테마 RS 시계열 추이 분석

**위치**: 52주 트렌드 페이지 차트 섹션

**사용법**

1. 분석할 테마를 하나 이상 선택합니다 (멀티셀렉 지원).
2. 분석 기간을 선택합니다: `1M` (1개월), `3M` (3개월), `6M` (6개월), `1Y` (1년).
3. 선택한 테마들의 RS 시계열 차트가 표시됩니다.

**차트 읽기**

- 상승 추세: 테마의 시장 내 강도가 지속적으로 증가 중
- 하락 추세: 테마의 강도가 약화되는 중
- 급등 후 조정: 단기 급등 이후 안정화 또는 하락

**멀티셀렉 활용**

여러 테마를 동시에 선택하면 테마 간 상대적 강도를 비교할 수 있습니다. 예를 들어 "AI"와 "반도체" 테마를 동시에 선택하면 두 테마의 강도 추이를 한 차트에서 비교할 수 있습니다.

### 5. 지속 강세 종목 탐지

**위치**: 52주 트렌드 페이지 지속 강세 종목 섹션

**사용법**

1. 분석 기간(`N일`)과 최소 등장 횟수(`M회`)를 설정합니다.
   - 예: "최근 20일 중 5회 이상"
2. 조회 버튼을 클릭합니다.
3. 조건을 만족하는 종목 목록이 테마별로 표시됩니다.

**컬럼 설명**

| 컬럼 | 설명 |
|------|------|
| 종목명 | 해당 종목 이름 |
| 테마 | 종목이 속한 테마 |
| 등장 횟수 | N일 내 신고가 리스트 등장 횟수 |
| 평균 RS | 등장 시 평균 RS 점수 |
| 등락률 변화 | 최근 등락률 추이 |

**활용 팁**

- 등장 횟수가 높을수록 지속적으로 강세를 유지하는 종목입니다.
- 등장 횟수가 높고 RS 점수도 높은 종목을 우선 주목하세요.

### 6. 그룹 액션 탐지

**위치**: 52주 트렌드 페이지 그룹 액션 탐지 섹션

**사용법**

1. 세 가지 파라미터를 슬라이더로 조절합니다.
2. 조건에 맞는 테마 목록이 실시간으로 업데이트됩니다.

**파라미터 설명**

**시간 윈도우 (1~7일)**

신규 테마 등장 여부를 판단하는 기간입니다.

- 값이 작을수록 (1~2일): 매우 최근에 처음 등장한 테마만 탐지
- 값이 클수록 (5~7일): 최근 수일 내에 등장한 테마까지 포함

각 파라미터 옆의 `?` 툴팁 아이콘에 마우스를 올리면 상세 설명을 볼 수 있습니다.

**RS 임계값 (-10 ~ +20)**

테마의 RS 상승 여부를 판단하는 기준값입니다.

- 0 이상: RS가 상승한 경우만 탐지
- 음수: RS가 소폭 하락해도 탐지 (더 많은 결과)
- 높은 양수: RS가 크게 상승한 경우만 탐지 (더 엄격한 필터)

**상태 임계값 (1~20)**

탐지된 테마의 상태를 분류하는 기준입니다.

| 상태 | 의미 |
|------|------|
| `new_theme` | 이전 데이터 없는 완전 신규 테마 |
| `new` | RS 변화가 임계값을 초과하는 강한 상승 |
| `returning` | RS 변화가 임계값 이하로 하락 |
| `neutral` | 그 외 중립 상태 |

**파라미터 조절 효과**

- 시간 윈도우 또는 RS 임계값 변경: 서버에 재요청하여 새로운 데이터 로드
- 상태 임계값 변경: 기존 데이터로 즉시 재계산 (서버 요청 없음)

### 7. 교집합 추천

**위치**: 52주 트렌드 페이지 교집합 추천 섹션

**사용법**

1. 교집합 추천 섹션으로 스크롤합니다.
2. 분석 날짜가 자동으로 선택됩니다 (양쪽 소스 모두 데이터가 있는 최신 날짜).
3. 교집합 테마와 해당 종목 목록이 표시됩니다.

**교집합 분석이란**

52주 신고가 데이터와 MTT 데이터, 두 독립적인 분석 방법론에서 **동시에** 등장하는 테마와 종목을 찾아냅니다.

두 방법론이 모두 강세로 판단한 종목은 신뢰도가 더 높습니다.

**2단계 분석**

- **Level 1 (테마 교집합)**: 두 소스에서 공통으로 나타나는 강세 테마
- **Level 2 (종목 교집합)**: 해당 테마 내에서 두 소스 모두에서 나타나는 종목

**컬럼 설명**

| 컬럼 | 설명 |
|------|------|
| 테마명 | 교집합 테마 이름 |
| 교집합 종목 수 | 두 소스 모두 등장하는 종목 수 |
| 평균 RS (52주) | 52주 신고가 기준 평균 RS |
| 평균 RS (MTT) | MTT 기준 평균 RS |

## FAQ

### Q1. RS 점수는 어떻게 계산되나요?

RS(상대강도) 점수는 해당 테마에 속한 종목들의 상대강도 지표를 집계한 값입니다. 구체적으로는 각 종목의 RS 점수를 합산하거나 평균내어 테마의 전체 강도를 표현합니다. 점수가 높을수록 해당 테마의 종목들이 시장 대비 강세를 보이고 있음을 나타냅니다.

### Q2. 데이터는 얼마나 자주 업데이트되나요?

데이터는 매 거래일마다 HTML 리포트 파일을 통해 업데이트됩니다. 서버가 실행 중일 때 새로운 HTML 파일이 데이터 폴더에 추가되면 자동으로 감지하여 데이터베이스에 적재합니다. 수동으로 업데이트하려면 백엔드의 동기화 기능을 사용하면 됩니다.

### Q3. 52주 신고가와 MTT 종목의 차이는 무엇인가요?

- **52주 신고가**: 최근 52주(약 1년) 내 가장 높은 가격을 달성한 종목 목록입니다. 모멘텀 투자에 활용됩니다.
- **MTT 종목**: Mark Minervini의 기술적 분석 템플릿(VCP 패턴 등)을 만족하는 종목 목록입니다. 기술적 진입점 탐색에 활용됩니다.

두 소스를 교차 분석하면 모멘텀과 기술적 패턴을 동시에 만족하는 고신뢰 종목을 발굴할 수 있습니다.

### Q4. 그룹 액션 탐지의 파라미터를 어떻게 설정해야 하나요?

초보자에게는 기본값(시간 윈도우: 3일, RS 임계값: 0, 상태 임계값: 5)을 권장합니다. 탐지 결과가 너무 많으면 RS 임계값을 높이고, 너무 적으면 낮추세요. 단기 신호에 집중하려면 시간 윈도우를 1~2일로 줄이세요.

### Q5. 교집합 추천에서 날짜가 자동으로 설정되는 이유는?

교집합 분석은 52주 신고가 데이터와 MTT 데이터 **두 가지 모두** 존재하는 날짜에서만 의미가 있습니다. 시스템이 두 소스 모두 데이터가 있는 가장 최근 날짜를 자동으로 선택하므로, 항상 유효한 교집합 결과를 보여줄 수 있습니다.

### Q6. 모바일에서도 사용할 수 있나요?

네, 대시보드는 모바일 최적화가 적용되어 있습니다. 모바일 환경에서는 좌측 상단의 햄버거 메뉴 아이콘을 탭하여 내비게이션 메뉴를 열 수 있습니다. 차트와 테이블은 가로 스크롤을 지원하여 작은 화면에서도 전체 데이터를 확인할 수 있습니다.

### Q7. API를 직접 호출해서 데이터를 사용할 수 있나요?

네, 백엔드 API를 통해 데이터에 직접 접근할 수 있습니다. API 문서는 서버 실행 후 http://localhost:8000/docs 에서 확인하거나, 프로젝트의 `API-DOCUMENTATION.md` 파일을 참조하세요.

## 문제 해결

### 페이지가 로드되지 않거나 데이터가 표시되지 않는 경우

**증상**: 대시보드에 접속했지만 데이터가 비어있거나 로딩이 멈춘 경우

**원인 및 해결 방법**

1. **백엔드 서버가 실행 중인지 확인**
   - 브라우저에서 http://localhost:8000/docs 에 접속해 보세요.
   - 접속이 안 된다면 백엔드 서버를 시작하세요.
   ```bash
   cd backend
   uv run uvicorn app.main:app --reload --port 8000
   ```

2. **데이터베이스에 데이터가 있는지 확인**
   - API를 통해 데이터 날짜 목록을 조회하세요.
   ```bash
   curl "http://localhost:8000/api/dates?source=52w_high"
   ```
   - 결과가 비어있다면 데이터 적재가 필요합니다.

3. **브라우저 캐시 초기화**
   - `Ctrl+Shift+R` (Windows/Linux) 또는 `Cmd+Shift+R` (Mac)로 강제 새로고침

### 데이터가 오래된 경우

**증상**: 최신 거래일 데이터가 보이지 않는 경우

**해결 방법**

1. 새 HTML 리포트 파일이 지정된 데이터 폴더에 있는지 확인합니다.
2. 백엔드 서버가 실행 중이라면 파일 감시 기능(Watchdog)이 자동으로 감지하여 적재합니다.
3. 서버를 재시작하면 수동 동기화가 실행됩니다.

```bash
# 서버 재시작 시 자동 동기화 실행
cd backend
uv run uvicorn app.main:app --reload --port 8000
```

### 교집합 추천에 결과가 없는 경우

**증상**: 교집합 추천 섹션에 "데이터 없음" 또는 빈 결과가 표시되는 경우

**원인 및 해결 방법**

1. **두 데이터 소스 모두 같은 날짜의 데이터가 필요합니다.**
   - 52주 신고가와 MTT 데이터가 같은 날짜에 모두 적재되어 있어야 합니다.
   - 날짜 목록을 각각 확인하세요.
   ```bash
   curl "http://localhost:8000/api/dates?source=52w_high"
   curl "http://localhost:8000/api/dates?source=mtt"
   ```

2. **날짜가 겹치지 않는 경우** 두 소스 모두의 HTML 파일을 적재한 후 다시 시도하세요.

### 차트가 표시되지 않는 경우

**증상**: 테마 RS 시계열 차트 영역이 비어있거나 에러가 표시되는 경우

**해결 방법**

1. 테마를 선택했는지 확인합니다 (차트는 테마 선택 후 표시됩니다).
2. 선택한 기간에 데이터가 있는지 확인합니다 (데이터가 적은 경우 기간을 줄여보세요).
3. 브라우저 콘솔(`F12 > Console`)에서 오류 메시지를 확인하고 지원팀에 문의하세요.

### 그룹 액션 탐지 결과가 항상 0개인 경우

**증상**: 파라미터를 조절해도 탐지 결과가 0개인 경우

**해결 방법**

1. RS 임계값을 낮춰보세요 (예: 0 → -5).
2. 시간 윈도우를 늘려보세요 (예: 3 → 7).
3. 해당 날짜에 충분한 데이터가 있는지 확인하세요.
4. 데이터가 충분한데도 결과가 없다면, 그 날짜에는 실제로 조건을 만족하는 테마가 없는 것일 수 있습니다.
